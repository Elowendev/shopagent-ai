# Claude Code Development Rules

你是“小拾光 Agent · ShopAgent AI”项目的 AI 开发助手。

你的任务：

帮助开发一个：

“面向原创品牌店铺的 AI 客服 Agent 系统”。

---

# 项目目标

当前项目：

不是普通聊天机器人。

而是：

具备：

* 商品推荐
* 品牌语气
* 用户意图识别
* Agent 决策能力

的 AI 店员系统。

---

# 当前阶段

当前阶段：

MVP（最小可用版本）。

优先：

* 可运行
* 可演示
* 架构清晰

不要：

* 过度工程化
* 提前优化
* 添加复杂基础设施

---

# 技术栈

前端：

* Next.js
* TailwindCSS
* Framer Motion

后端：

* FastAPI

AI：

* DeepSeek API

Agent：

* LangGraph

数据库：

* Supabase

---

# UI 原则

UI 必须：

* 极简
* 安静
* 有留白
* 有品牌感
* 不像 SaaS Dashboard

禁止：

* 花哨渐变
* 强科技蓝
* 复杂卡片堆叠
* 游戏化 UI

---

# 品牌语气规则

AI 回复必须：

* 温柔
* 自然
* 日常感
* 像真正店员

禁止：

* “亲亲”
* “宝子”
* “姐妹”
* 过度营销
* 情绪讨好

---

# 代码原则

## 原则 1

优先：

可读性。

---

## 原则 2

优先：

简单架构。

---

## 原则 3

避免：

过度抽象。

---

## 原则 4

每次只完成：

一个明确功能。

---

# 文件结构原则

保持：

清晰、简单。

不要：

过度拆分。

---

# Agent 原则

当前阶段：

不要构建复杂 Autonomous Agent。

而是：

Focused Agent。

即：

围绕：

* 商品推荐
* 客服回复

进行明确任务处理。

---

# 回复原则

当生成代码时：

1. 先解释要做什么
2. 再输出代码
3. 不一次生成过大文件
4. 优先生成可运行版本

---

# 当前优先级

当前最重要的是：

1. AI 对话界面
2. 商品 JSON 读取
3. DeepSeek API 接入
4. 商品推荐逻辑
5. 品牌语气生成

---

# 当前禁止开发

暂时不要开发：

* 登录系统
* 权限系统
* Docker
* Kubernetes
* 微服务
* 多 Agent 系统
* 自动部署

---

# 开发理念

这是：

一个真实业务 AI Agent 项目。

不是：

AI Demo。

所有设计：

优先考虑：

“真实店铺是否真的会使用。”

---

# 商品数据规则

商品数据：

全部使用中文 JSON。

路径：

/products

规则：

* 一个商品一个 JSON 文件
* 风格标签必须具备语义
* 强调情绪与场景
* 不使用机械参数化描述
* description 必须具备画面感
* brand_voice_description 必须像真实店主说话

---

# AI 回复规则

AI 回复：

更像：

“懂审美的店员”。

而不是：

“标准客服”。

回复需要：

* 自然
* 有温度
* 有审美理解
* 不机械

但不要：

* 过度人格化
* 情绪陪伴化
* AI 女友化
